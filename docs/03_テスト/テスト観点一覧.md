## 1. 文書概要

- 対象ブランチ: `develop`
- 目的: 次フェーズで追加・更新した仕様に対するテスト観点を整理する
- 対象範囲:
  - コメント機能
  - 添付ファイル機能
  - 通知一覧 / 未読通知件数
  - アクティビティログ
  - タスク論理削除
  - 認可制御
  - 通知ポーリング

---

## 2. テスト分類

| 分類 | 内容 |
|---|---|
| 正常系 | 想定通りの入力・権限・状態で成功すること |
| 異常系 | 入力不正、対象未存在、権限不足、認証切れで適切に失敗すること |
| 境界値 | 文字数上限、件数上限、サイズ上限などの境界条件を満たすこと |
| 連携 | DB、S3、通知、履歴、画面表示が整合すること |
| 画面制御 | ボタン表示制御、メッセージ表示、遷移が正しいこと |

## 3. コメント機能

## 3.1 コメント一覧取得

| No | 観点 | 確認内容 |
|---:|---|---|
| CMT-L-01 | 正常取得 | 対象タスクのコメント一覧が取得できること |
| CMT-L-02 | 並び順 | `createdAt ASC, id ASC` で表示されること |
| CMT-L-03 | 論理削除除外 | 削除済みコメントが一覧に表示されないこと |
| CMT-L-04 | 0件表示 | コメント0件時に空状態メッセージが表示されること |
| CMT-L-05 | ページング | `page`, `size` に応じて取得結果が切り替わること |
| CMT-L-06 | 認可 | 対象タスクを参照できない場合は取得できないこと |
| CMT-L-07 | 対象未存在 | タスク未存在時に適切なエラーとなること |
| CMT-L-08 | プレースホルダ非表示 | 削除済みコメントを `削除済み` 表示へ置き換えず、一覧から除外すること |

| CMT-C-09 | 本人除外 | 操作者本人には通知されないこと |

## 12. 実施優先度

| 優先度 | 対象 |
|---|---|
| 高 | タスク論理削除、コメント投稿/更新/削除、添付アップロード/削除、通知既読化、未読通知件数 |
| 中 | 通知一覧、履歴一覧、S3補償処理、通知ポーリング |
| 低 | バッジ99+表示、タブ非アクティブ時制御、細かな表示文言 |
