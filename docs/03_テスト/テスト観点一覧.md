## 1. 文書概要

- 対象ブランチ: `develop`
- 目的: 現行仕様の主要機能に対するテスト観点を統合的に整理する
- 対象範囲:
  - 認証、セッション、ログアウト
  - タスク一覧、作成、詳細、更新、論理削除
  - 担当者候補一覧
  - コメント機能
  - 添付ファイル機能
  - 通知一覧、未読通知件数、既読化、通知ポーリング
  - アクティビティログ
  - 認可制御、画面表示制御
- 注記:
  - ADMIN に関する観点は将来拡張時の確認観点として記載する
  - 現行実装の必須テスト対象は MEMBER 前提の観点とする

---

## 2. テスト分類

| 分類 | 内容 |
|---|---|
| 正常系 | 想定通りの入力・権限・状態で成功すること |
| 異常系 | 入力不正、対象未存在、権限不足、認証切れで適切に失敗すること |
| 境界値 | 文字数上限、件数上限、サイズ上限などの境界条件を満たすこと |
| 連携 | DB、S3、通知、履歴、画面表示が整合すること |
| 画面制御 | ボタン表示制御、メッセージ表示、遷移が正しいこと |
