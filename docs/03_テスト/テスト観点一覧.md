## 1. 文書概要

- 対象ブランチ: `develop`
- 目的: 現行仕様の主要機能に対するテスト観点を統合的に整理する
- 対象範囲:
  - 認証、セッション、ログアウト
  - タスク一覧、作成、詳細、更新、論理削除
  - 担当者候補一覧
  - コメント機能
  - 添付ファイル機能
  - 通知一覧、未読通知件数、既読化、通知ポーリング
  - アクティビティログ
  - 認可制御、画面表示制御
  - チーム作成、所属チーム一覧、チーム詳細
  - チームメンバー一覧、追加候補ユーザー一覧
  - チームメンバー追加、ロール変更、メンバー削除
  - チーム内ロールによる認可制御（OWNER / ADMIN / MEMBER）
  - 所属全チーム横断タスク一覧
  - チーム文脈付きタスク一覧
  - チーム所属前提でのタスク作成 / 詳細 / 更新 / 削除
  - 既存データ移行後の確認
  - チーム管理操作の監査ログ
  - チーム管理機能に伴う並行操作確認
- 注記:
  - 本フェーズでは、チーム内ロール `OWNER` / `ADMIN` / `MEMBER` をテスト対象とする
  - 既存のシステム全体ロールとチーム内ロールは分離して扱う
  - チーム削除、OWNER移譲、招待機能は本フェーズ対象外とする

## 2. テスト分類

| 分類 | 内容 |
|---|---|
| 正常系 | 想定通りの入力・権限・状態で成功すること |
| 異常系 | 入力不正、対象未存在、権限不足、認証切れで適切に失敗すること |
| 境界値 | 文字数上限、件数上限、サイズ上限、候補0件、データ0件などの境界条件を満たすこと |
| 連携 | DB、S3、通知、履歴、ログ、認可、画面表示が整合すること |
| 画面制御 | ボタン表示制御、メッセージ表示、遷移が正しいこと |
| 移行確認 | 既存データ移行後の `team_id`、初期チーム、メンバー所属、OWNER制約が正しいこと |
| 並行操作 | ロール変更競合、メンバー追加 / 削除競合、タスク更新とメンバー削除競合で整合が崩れないこと |
