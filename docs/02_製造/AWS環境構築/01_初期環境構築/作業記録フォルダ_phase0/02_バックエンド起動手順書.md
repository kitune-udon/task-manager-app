# バックエンド起動手順書

## 前提

- JDK 17 が利用できること
- Docker が利用できること
- PostgreSQL コンテナを起動できること
- 作業ディレクトリがリポジトリ直下であること

## 使用技術

- Spring Boot 3.5.13
- Gradle Wrapper 8.14.4
- PostgreSQL 15
- Flyway

## 基本手順

### 1. DB を起動する

```bash
docker compose -f backend/docker-compose.yml up -d db
```

今回確認結果:

- `task-db` が `healthy` で起動
- `0.0.0.0:5432->5432/tcp` で待受

### 2. テストで依存関係と DB 接続を確認する

```bash
cd backend
./gradlew test
```

今回確認結果:

- `BUILD SUCCESSFUL`
- `TaskApplicationTests` 通過

### 3. アプリを起動する

```bash
cd backend
./gradlew bootRun
```

今回確認結果:

- Tomcat が `8080` で初期化
- HikariCP により `jdbc:postgresql://localhost:5432/taskdb` へ接続
- Flyway が `3 migrations` を検証
- `Schema "public" is up to date. No migration necessary.`
- `Started TaskApplication` まで到達

## 起動確認ポイント

- ログに `Tomcat initialized with port 8080` が出る
- ログに `HikariPool-1 - Start completed.` が出る
- ログに `Current version of schema "public": 3` が出る
- `/actuator/health` が有効化されている

## 既知注意点

- アクティブプロファイル未指定時は `default` で起動する
- DB が起動していないと bootRun/test で接続失敗する
- 連続で再起動すると、プロセス解放待ちのタイミングで `Port 8080 is already in use` が出る場合がある
