# 正常系E2Eシナリオ一覧

## 目的

- 初回 AWS 公開対象の正常系ユーザーフローを、E2E シナリオとして独立管理する
- Playwright で自動化している内容を、公開判定や保守で参照しやすい形にする
- ローカル観点表のうち、正常系 E2E として担保している範囲を明確にする

## 前提

- PostgreSQL が起動している
- backend が起動している
- frontend が起動している
- `backend` / `frontend` の環境変数が設定済みである
- 初回公開対象外のコメント、添付、パスワード再発行はシナリオ対象外とする

## 実行方法

- 実行コマンド: `cd frontend && npm run test:e2e`
- 実装ファイル:
  - `frontend/playwright.config.ts`
  - `frontend/tests/e2e/helpers.ts`
  - `frontend/tests/e2e/mvp-flow.spec.ts`
- 2026-04-08 実行結果: `3 passed`

## シナリオ詳細

### E2E-01 新規登録成功

1. 新規ユーザー情報を生成する
2. 新規登録画面で必須項目を入力する
3. 登録ボタンを押す
4. ログイン画面へ戻ることを確認する

### E2E-02 ログイン成功

1. E2E 用に登録したユーザーでログインする
2. タスク一覧画面へ遷移したことを確認する

### E2E-03 タスク作成成功

1. 一覧画面からタスク作成へ進む
2. タイトル、説明、期限を入力する
3. 担当者候補が存在する場合は選択する
4. 作成ボタンを押す
5. 一覧へ戻り、対象タスクの行が表示されることを確認する

### E2E-04 タスク詳細表示成功

1. 一覧から作成済みタスクの詳細を開く
2. タイトルなどの内容が表示されることを確認する

### E2E-05 タスク更新成功

1. 詳細画面から編集へ進む
2. タイトル、ステータス、優先度を変更する
3. 更新ボタンを押す
4. 詳細画面へ戻り、変更後の内容が表示されることを確認する

### E2E-06 タスク削除成功

1. 詳細画面で削除ボタンを押す
2. 確認ダイアログを承諾する
3. 一覧へ戻り、削除したタスクが消えていることを確認する

## 補足

- 現在の `mvp-flow.spec.ts` には、正常系に加えて以下の異常系/セッション系も含まれている
  - ログイン失敗がログイン画面内で扱われること
  - セッション切れ時にログイン画面へ戻されること
- 異常系・境界値の専用成果物は別タスク `0.2-6` として扱う

## 公開判定での使い方

- 本ファイルは「正常系の主要導線が自動実行で担保されている」ことを示す根拠として使う
- デプロイ後の最低限確認は `12_デプロイ後スモークテスト項目一覧.md` を使う
- 未実装候補や公開対象外機能の整理は `09_MVP公開範囲確定メモ.md` を正とする
