# 異常系・境界値シナリオ一覧

## 目的

- 初回 AWS 公開対象機能の異常系と境界値を、専用のシナリオ一覧として管理する
- backend 統合テストと frontend Playwright で担保している異常系を可視化する
- 自動化済みの範囲と、今後手動または追加自動化が必要な範囲を切り分ける

## 前提

- PostgreSQL が起動している
- backend が起動している
- frontend が起動している
- `backend` / `frontend` の環境変数が設定済みである
- コメント、添付、パスワード再発行は初回公開対象外として扱う

## 実行根拠

### backend 統合テスト

- 実装ファイル: `backend/src/test/java/com/example/task/MvpApiIntegrationTests.java`
- 主に API の 400 / 401 / 403 と認可境界を担保している

### frontend Playwright

- 実装ファイル: `frontend/tests/e2e/mvp-flow.spec.ts`
- 主に画面上の異常導線とセッション切れ時の遷移を担保している

## シナリオ詳細

### NEG-01 ログイン失敗

1. 事前にユーザーを登録する
2. ログイン画面で誤ったパスワードを入力する
3. ログインボタンを押す
4. ログイン画面に留まり、認証失敗メッセージが表示されることを確認する
5. セッション切れ用のメッセージが出ないことを確認する

### NEG-02 未認証アクセス

1. token なしで `GET /api/tasks` を呼ぶ
2. 401 と `AUTH-001` が返ることを確認する

### NEG-03 不正 token

1. `Authorization: Bearer invalid-token` で `GET /api/tasks` を呼ぶ
2. 401 と `AUTH-003` が返ることを確認する

### NEG-04 セッション切れ

1. 正常ログインする
2. localStorage 上の token を不正値へ書き換える
3. 画面を再読込する
4. ログイン画面へ戻り、再ログインメッセージが出ることを確認する

### NEG-05 新規登録の入力値不正

1. `name` を空、`email` を不正形式、`password` を短い値で登録する
2. 400 と `VAL-INPUT-001` が返ることを確認する
3. `details` に各項目の情報が含まれることを確認する

### NEG-06 タスク作成の必須入力不足

1. ログイン後、`title` を空のまま作成 API を呼ぶ
2. 400 と `VAL-TASK-001` が返ることを確認する

### NEG-07 非参照対象タスクの非表示

1. 自分が作成者/担当者であるタスクと、どちらでもないタスクを用意する
2. 一覧 API を呼ぶ
3. 自分が参照可能なタスクだけが返ることを確認する

### NEG-08 他人タスク参照不可

1. 対象外タスクの詳細 API を呼ぶ
2. 403 と `AUTH-005` が返ることを確認する

### NEG-09 他人タスク更新不可

1. 対象外タスクの更新 API を呼ぶ
2. 403 と `PERM-TASK-403-UPD` が返ることを確認する

### NEG-10 他人タスク削除不可

1. 対象外タスクの削除 API を呼ぶ
2. 403 と `PERM-TASK-403-DEL` が返ることを確認する

## 補足

- `MVP-05` のうち、frontend 画面上の項目別表示まではまだ専用 E2E で自動化していない
- `MVP-09` は backend API レベルでは自動化済みだが、画面ルーティング込みの確認は手動または追加 E2E の余地がある
- `MVP-15` のうち、タイトル 100 文字超や日付不正の画面経由確認は追加余地がある

## 今後の追加候補

- 新規登録の確認用パスワード未入力/不一致の Playwright 化
- 未認証で `/tasks` を直接開いた場合の画面遷移 E2E
- タイトル 100 文字超の frontend / backend 横断確認
- 一覧フィルタと空状態表示の E2E
