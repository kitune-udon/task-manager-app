# ローカルMVP通し確認観点表

## 目的

- MVP として公開候補にしている機能が、ローカル環境で一連操作できるかを確認する
- 成功/失敗/条件付きの 3 区分で記録し、公開可否判断の材料を残す
- 未実装機能や暫定 UI を MVP 対象に含めるかどうかの判断材料を作る

## 前提

- `backend` が起動している
- `frontend` が起動している
- PostgreSQL が起動している
- ブラウザの DevTools で Network / Console を確認できる

## 推奨テストデータ

| データ | 用途 | 備考 |
| --- | --- | --- |
| ユーザーA | 基本操作確認 | 自分で作成したタスクを操作する |
| ユーザーB | 認可確認 | 他人タスクへの参照/更新/削除確認に使う |
| タスクA | ユーザーA作成・A担当 | 詳細/編集/削除の正常系に使う |
| タスクB | ユーザーA作成・B担当 | 認可境界の確認に使う |
| タスクC | 期限なし・担当者なし | 一覧/詳細表示の欠損値確認に使う |

## ステータス記録ルール

- `OK`: 期待どおりに完了
- `NG`: 期待どおりに動作しない
- `条件付き`: 一部制約あり、または公開前判断が必要
- `未確認`: まだ実施していない
