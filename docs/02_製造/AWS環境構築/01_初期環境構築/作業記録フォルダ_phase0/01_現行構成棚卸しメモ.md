# 現行構成棚卸しメモ

## システム概要

- バックエンド: Spring Boot 3.5.13 / Java 17 / Spring Security / Spring Data JPA / Flyway / PostgreSQL / JWT
- フロントエンド: React 19 / TypeScript / Vite 8 / Axios
- DB: PostgreSQL 15 を `backend/docker-compose.yml` で起動
- 認証方式: メールアドレス + パスワードでログインし、JWT を Bearer Token として利用

## 現行ディレクトリの役割

- `backend/src/main/java/com/example/task/controller`
  - 認証 API、タスク API、担当者候補 API を提供
- `backend/src/main/java/com/example/task/service`
  - 認証、タスク、ユーザー候補取得の業務処理
- `backend/src/main/resources`
  - Spring 設定と Flyway マイグレーション
- `frontend/src/pages`
  - ログイン、新規登録、タスク一覧、作成、詳細、編集の各画面
- `frontend/src/lib`
  - API クライアント、認証 API、タスク API、担当者候補 API

## 実装済みの主要機能

- 認証
  - `POST /api/auth/register`
  - `POST /api/auth/login`
  - `GET /api/auth-test/me`
- タスク
  - `POST /api/tasks`
  - `GET /api/tasks`
  - `GET /api/tasks/{taskId}`
  - `PUT /api/tasks/{taskId}`
  - `DELETE /api/tasks/{taskId}`
- 担当者候補
  - `GET /api/users`

## 画面観点の現状

- 実装済み
  - ログイン
  - 新規登録
  - タスク一覧
  - タスク作成
  - タスク詳細
  - タスク編集
- 暫定/ダミー要素あり
  - コメント機能: フロント画面上の入力 UI のみで、バックエンド API は未実装
  - 添付ファイル機能: フロント画面上の表示/UI のみで、保存/取得 API は未実装

## 認証・認可の現状

- 認証
  - `/api/auth/**` は未認証で利用可能
  - それ以外の API は認証必須
- 認可
  - タスク一覧: 作成者または担当者のタスクのみ取得
  - タスク詳細: 作成者または担当者のみ参照可能
  - タスク更新: 作成者または担当者のみ可能
  - タスク削除: 作成者のみ可能
  - チーム ADMIN 条件は未実装

## 確認結果

- `backend`
  - `./gradlew test` 成功
  - `./gradlew bootRun` で Tomcat 8080 起動、DB 接続、Flyway version 3 適用済みを確認
