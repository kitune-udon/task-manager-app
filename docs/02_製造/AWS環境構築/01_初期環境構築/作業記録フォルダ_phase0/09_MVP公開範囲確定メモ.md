# MVP公開範囲確定メモ

## 目的

- 初回 AWS 公開で利用者に見せる機能範囲を確定する
- 未実装または暫定 UI を MVP 対象から切り分ける
- 公開判定のための自動テストと手動スモークの境界を明確にする

## 判断の前提

- `backend` の `./gradlew test` は成功済み
- `frontend` の `npm run build` は成功済み
- `frontend` の `npm run test:e2e` は `3 passed`
- `08_ローカルMVP通し確認観点表.md` の自動化済み観点をベースに判断する

## 初回 AWS 公開で含める機能

### 利用者向け機能

- 新規登録
- ログイン
- ログアウト
- セッション切れ時の再ログイン導線
- タスク一覧表示
- タスク一覧のステータス/優先度フィルタ
- タスク詳細表示
- タスク作成
- タスク編集
- タスク削除
- 担当者選択プルダウン

### API/業務ルール

- `GET /api/users` による担当者候補取得
- 作成者/担当者ベースのタスク参照/更新権限
- 作成者のみ削除可能
- 入力バリデーション
- 保護 API の 401 と auth API の 401 の分離

## 初回 AWS 公開から外す機能

### 明示的に除外するもの

- コメント投稿/保存/再表示
- 添付ファイルのアップロード/ダウンロード
- チーム ADMIN を含む最終版認可ルール
- パスワード再発行/パスワード忘れ導線

### 除外理由

- コメント/添付は backend 実装が未着手で、現状はダミー UI に留まる
- チーム ADMIN 判定はチーム機能未実装のため段階実装中
- パスワード再発行は画面上にボタンはあるが、処理が未実装

## 公開前の必須対応

### 優先度 高: MVP対象外UIの露出整理

- 対応済み: タスク詳細画面のコメント UI を非表示化
- 対応済み: タスク詳細画面の添付ファイル UI を非表示化
- 対応済み: ログイン画面の `パスワードを忘れた場合` ボタンを非表示化

### 優先度 高: 環境依存設定

- 対応済み: backend の DB 接続情報、JWT secret、CORS origin を環境変数化
- 対応済み: frontend の API Base URL を `VITE_API_BASE_URL` 化
- 対応済み: `backend/.env.example` と `frontend/.env.example` を追加

### 優先度 中

- `application-local.yml` / `application-prod.yml` の責務分離方針を決める
- `.env.example` などのサンプル設定を作る

## 公開判定のための検証方針

### 自動テストを必須とするもの

- `cd backend && ./gradlew test`
- `cd frontend && npm run build`
- `cd frontend && npm run test:e2e`

### 手動スモークとして残すもの

- ログアウト後に保護画面へ戻れないこと
- 未認証で `/tasks` を直接開いたときの導線
- 一覧フィルタ UI の見た目と操作感
- 空状態表示
- 編集画面の初期値反映
- 主要画面の文言、導線、レイアウト崩れ

## 公開可否の現時点判断

### 判定

- `条件付きで公開可能`

### 理由

- 認証、タスク CRUD、401 ハンドリングの主要フローは自動テストで担保できている
- MVP 対象外の UI 露出は整理済みで、初回公開スコープが明確になった
- 環境依存設定は環境変数ベースに寄せたが、AWS への実投入手順と profile 分離はまだ残っている

## Phase1 への申し送り

- まずは MVP 対象外 UI の露出整理を優先する
- 次に環境変数化と AWS 向け設定分離を行う
- その後に RDS / Elastic Beanstalk / S3/CloudFront の順で進める
- コメント/添付/チーム ADMIN は初回公開後の後続スコープとして管理する
