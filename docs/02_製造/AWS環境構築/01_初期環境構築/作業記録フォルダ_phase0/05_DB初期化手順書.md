# DB初期化手順書

## 初期化方式

- 採用方式: Flyway
- 有効設定: `spring.flyway.enabled: true`
- マイグレーション配置: `classpath:db/migration`

## 適用対象ファイル

1. `V1__init_schema.sql`
   - `users` テーブル作成
   - `tasks` テーブル作成
   - `assigned_user_id` 外部キー作成
2. `V2__change_due_date_to_date.sql`
   - `tasks.due_date` を `DATE` 化
3. `V3__add_created_by_to_tasks.sql`
   - `tasks.created_by` 追加
   - 既存データ backfill
   - `created_by` 外部キーと index 作成

## 現在の主要テーブル

- `users`
- `tasks`

## 初期化手順

1. PostgreSQL コンテナを起動する
2. backend を `bootRun` で起動する
3. 起動時に Flyway が migration を検証・適用する

## 今回確認結果

- backend 起動ログで以下を確認
  - `Successfully validated 3 migrations`
  - `Current version of schema "public": 3`
  - `Schema "public" is up to date. No migration necessary.`

## 初期データの扱い

## 注意点

- `V3__add_created_by_to_tasks.sql` は既存 task の `created_by` を backfill する
- task が存在し、かつ users が 1 件もない状態では V3 が失敗する可能性がある
