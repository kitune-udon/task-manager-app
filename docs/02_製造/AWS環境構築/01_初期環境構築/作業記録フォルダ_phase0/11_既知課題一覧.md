# 既知課題一覧

## 目的

- 公開前に必ず塞ぐものと、公開後対応へ回すものを明確に分ける
- 不具合、暫定実装、未実装機能、確認不足を 1 つの一覧で追えるようにする
- Phase1 以降の優先順位付けに使える状態にする

## 優先度定義

- `Must`: 初回 AWS 公開または Phase2 着手前に対応が必要
- `Should`: できれば公開前に確認・対応したいが、代替策がある
- `Later`: 初回公開後の後続スコープとして扱う

## 公開判断メモ

- 初回公開の前提では、`Later` の項目は MVP 対象外として扱う
- `Must` が残ったままでは Phase2 以降の AWS 実作業に入らない
- `Should` は公開前に確認するのが望ましいが、手順と代替策が残っていれば管理可能
