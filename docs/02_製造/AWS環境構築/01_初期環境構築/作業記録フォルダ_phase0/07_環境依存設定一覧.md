# 環境依存設定一覧

## 調査結果サマリ

- backend / frontend ともに実運用向けの `.env.example` を作成済み
- frontend は `.env.local` / `.env.production` を作成済み
- frontend は `import.meta.env.VITE_API_BASE_URL` から API 接続先を読める
- backend は `application.yml` / `application-dev.yml` のプレースホルダ経由で環境変数を読める
- backend の CORS origin は `app.cors.allowed-origins` と `CorsProperties` で外部化済み
- ローカル未設定時は従来どおり localhost 向け既定値へフォールバックする

## 設定読込箇所の補足

- backend
  - `JwtProperties` は `@ConfigurationProperties(prefix = "app.jwt")` で読込
  - `CorsProperties` は `@ConfigurationProperties(prefix = "app.cors")` で読込
  - datasource は Spring Boot 標準の `spring.datasource.*` に環境変数プレースホルダを設定
- frontend
  - `axios.create({ baseURL: API_BASE_URL })` の `API_BASE_URL` は `import.meta.env.VITE_API_BASE_URL` から解決
  - `frontend/.env.example` を配置済み

## 実装済みの環境変数

- backend
  - `DB_URL`
  - `DB_USERNAME`
  - `DB_PASSWORD`
  - `JWT_SECRET`
  - `JWT_EXPIRATION_MILLIS`
  - `CORS_ALLOWED_ORIGINS`
- frontend
  - `VITE_API_BASE_URL`

## 追加メモ

- backend には `application-local.yml` / `application-prod.yml` を追加済み
- frontend には `.env.local` / `.env.production` を追加済み
- frontend の `.env.production` は placeholder なので、本番 backend URL が確定したら差し替える
- frontend / backend ともに `.env.example` は追加済み
- 本番対応前に「共通設定」「local」「prod」の責務分離を継続整理する
