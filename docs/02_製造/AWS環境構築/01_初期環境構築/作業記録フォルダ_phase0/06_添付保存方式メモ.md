# 添付保存方式メモ

## 結論

- 現時点で、添付ファイル機能は backend 実装が存在しない
- フロント画面上に添付ファイル UI はあるが、動的保存/取得/削除は行っていない
- AWS デプロイ前に「一時停止」か「S3 前提で後続実装」かを決める必要がある

## backend 調査結果

- `MultipartFile`
- `upload`
- `download`
- `attachment`

上記キーワードで backend 実装は未検出

## frontend 調査結果

- `TaskDetailPage.tsx` に以下の暫定 UI がある
  - コメント表示
  - コメント入力欄
  - `コメント投稿` ボタン
  - `添付ファイル` 表示欄
  - `DL` ボタン
  - `ファイルを添付` ボタン

ただし現状は:

- コメント本文に task の説明文を流用
- 添付ファイル名に `task-{id}.xlsx` を擬似表示
- API 連携なし
- 保存先なし
- 権限制御なし

## 本番化観点の判断

- A. 一時停止
  - MVP 公開優先なら最も安全
  - 画面から導線を外す/非表示にする方針が取りやすい
- B. 暫定ローカル保存
  - AWS では再デプロイやスケール時に破綻しやすく非推奨
- C. S3 前提で実装
  - 本番でも添付を使うなら最終的にはこの方針が必要

## Phase1 への申し送り

- 添付を MVP 対象から外すなら、画面/設計書/API の露出も揃えて止める
- 添付を残すなら、Phase5 の S3 化を見据えた設計が必要
- コメント機能も同様に backend 実装未着手のため、MVP 対象可否の判断が必要
