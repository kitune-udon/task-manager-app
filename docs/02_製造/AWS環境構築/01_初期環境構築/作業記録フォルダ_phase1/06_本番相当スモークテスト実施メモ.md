# 本番相当スモークテスト実施メモ

## 目的

- Phase1 で実施した設定分離や MVP 露出整理の結果を、ローカルで本番相当の形で確認する
- backend / frontend / E2E の主要確認結果を 1 つの成果物にまとめる
- Phase1 完了判定の前提情報を残す

## 実施日時

- 2026年4月8日

## 実施コマンド

### backend

- `cd backend && ./gradlew test`

### frontend

- `cd frontend && npm run build`
- `cd frontend && npm run test:e2e`

## 結果

### backend

- `./gradlew test` 成功
- 出力:
  - `BUILD SUCCESSFUL in 2s`
  - `:test UP-TO-DATE`

### frontend build

- `npm run build` 成功
- 出力:
  - `vite v8.0.3 building client environment for production...`
  - `✓ built in 356ms`

### frontend E2E

- `npm run test:e2e` 成功
- 結果:
  - `3 passed (12.1s)`
- 確認できたシナリオ:
  - ログイン失敗が画面内エラーで扱われること
  - タスクの作成、更新、削除が画面操作で完了できること
  - セッション切れ時にログイン画面へ戻されること

## 本番相当確認として扱う理由

- backend は profile 分離後に `local` / `prod` の両方で起動確認済み
- frontend は `.env.local` / `.env.production` を用意し、`build` と `preview` で確認済み
- MVP の主要導線は Playwright により継続確認できる状態

## 補足

- `frontend/.env.production` は placeholder の API URL を使って build している
- そのため、本番 backend の実 URL に対する疎通確認は AWS デプロイ後スモークで別途行う
- 今回のメモは、ローカルで実施可能な本番相当確認までを対象とする

## 判定

- `1.4-3 ローカルで本番相当スモークテストを実施する` は対応済み
