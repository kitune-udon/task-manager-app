# 添付ファイル本番方針決定メモ

## 目的

- 添付ファイル機能の本番方針を `A / B / C` から 1 つに確定する
- 初回 AWS 公開時点での扱いと、後続実装の切り分けを明確にする
- UI 非表示対応と S3 移行検討の関係を整理する

## 実施日時

- 2026年4月8日

## 判断対象

- A. 一時停止
- B. 暫定ローカル保存
- C. S3 前提で実装

## 現状整理

- backend に添付アップロード/保存/取得/削除 API は存在しない
- frontend の添付 UI は本来ダミー表示であり、MVP 対象外として非表示化済み
- 初回 AWS 公開の MVP 範囲から、コメント/添付は除外済み
- 本番環境でのローカル保存は、再デプロイやスケール時の整合性が保てない

## 決定

- 初回 AWS 公開時点では `A. 一時停止` を採用する

## 採用理由

- 現時点で backend 実装が未着手であり、MVP 公開の対象機能ではない
- 暫定ローカル保存は AWS 運用と相性が悪く、後で作り直しになりやすい
- 本番で添付を使うなら、最終的には S3 前提の設計に寄せる必要がある
- まずはタスク管理の主要導線を安定公開する方が優先度が高い

## 初回公開時の扱い

- 添付ファイル機能は公開対象に含めない
- 画面上の添付 UI は非表示状態を維持する
- API 設計、AWS 構成、運用確認の対象から添付アップロード処理を外す

## 後続対応

- 添付機能を再開する場合は `C. S3 前提で実装` を採用候補とする
- その際は、保存先、メタ情報、権限制御、削除方針を合わせて設計する
- Phase1 では次タスクとして、S3 移行を見据えた DB 項目整理を行う

## 判定

- `1.3-2 本番方針を A / B / C から 1 つに決定する` は対応済み
