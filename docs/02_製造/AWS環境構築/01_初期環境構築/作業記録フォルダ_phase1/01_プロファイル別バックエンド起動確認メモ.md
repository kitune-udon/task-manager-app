# プロファイル別バックエンド起動確認メモ

## 目的

- `application-local.yml` と `application-prod.yml` を使った起動確認結果を記録する
- AWS 向け設定分離後も、プロファイル切替で backend が起動できることを確認する

## 実施日時

- 2026年4月8日

## 実施内容

### local プロファイル

- 実行コマンド:
  - `GRADLE_USER_HOME=/tmp/gradle-home SERVER_PORT=18080 ./gradlew bootRun --args='--spring.profiles.active=local'`
- 確認ポイント:
  - active profile が `local` になること
  - PostgreSQL に接続できること
  - Flyway validation が通ること
  - Tomcat が起動すること
  - `/actuator/health` が `UP` を返すこと
- 結果:
  - 起動成功
  - `Tomcat started on port 18080`
  - `The following 1 profile is active: "local"`
  - `http://127.0.0.1:18080/actuator/health` が `UP`

### prod プロファイル

- 実行コマンド:
  - `GRADLE_USER_HOME=/tmp/gradle-home SERVER_PORT=18081 ./gradlew bootRun --args='--spring.profiles.active=prod'`
- 確認ポイント:
  - active profile が `prod` になること
  - PostgreSQL に接続できること
  - Flyway validation が通ること
  - Tomcat が起動すること
  - `/actuator/health` が `UP` を返すこと
- 結果:
  - 起動成功
  - `Tomcat started on port 18081`
  - `The following 1 profile is active: "prod"`
  - `http://127.0.0.1:18081/actuator/health` が `UP`

## 補足

- 既存の Java プロセスが `8080` を使用していたため、確認時は `SERVER_PORT` を上書きした
- `8080` の LISTEN は `java` プロセス `PID 26136` が保持していた
- DB 接続先は既定値の `jdbc:postgresql://localhost:5432/taskdb` を使用した

## 判定

- `1.1-8 プロファイル別にバックエンド起動確認を行う` は対応済み
