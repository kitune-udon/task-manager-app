<summary>2. エラー設計方針</summary>

| 観点 | 方針 | 理由 |
|---|---|---|
| 分類 | 業務エラーとシステムエラーを分離する | ユーザー向け通知と障害調査を分離するため |
| 表現 | HTTPステータス + エラーコードを返す | フロント、ログ、運用で共通言語化するため |
| 追跡性 | requestId を全エラーで返却する | フロント問い合わせとサーバーログを突合しやすくするため |
| セキュリティ | 内部情報を出しすぎない | 攻撃者に余計な情報を与えないため |
| 入力エラー | field単位で details を返す | UIで項目エラー表示しやすくするため |
| システムエラー | 利用者向けメッセージは汎用化する | 内部構成や例外詳細を秘匿するため |

<summary>3. エラーレスポンス共通仕様</summary>

## 3.1 共通レスポンス形式

```json
{
  "timestamp": "2026-04-13T22:00:00+09:00",
  "status": 400,
  "errorCode": "ERR-INPUT-001",
  "message": "入力内容に誤りがあります",
  "details": [
    {
      "field": "title",
      "message": "タイトルを入力してください。"
    }
  ],
  "path": "/api/tasks",
  "requestId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

## 3.2 項目定義

| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| timestamp | string | ○ | エラー発生日時 |
| status | number | ○ | HTTPステータス |
| errorCode | string | ○ | アプリケーションエラーコード |
| message | string | ○ | 利用者向けエラーメッセージ |
| details | array | △ | 項目単位のエラー詳細。主にバリデーションエラー時に返却 |
| path | string | ○ | リクエストURI |
| requestId | string | ○ | リクエスト相関ID |

## 3.3 details 項目定義

| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| field | string | ○ | エラー対象フィールド名 |
| message | string | ○ | 項目別メッセージ |

## 3.4 共通方針

- `details` は入力エラー時のみ付与されることがある
- `requestId` はレスポンス本文とレスポンスヘッダ `X-Request-Id` の両方で返却する
- `path` には対象APIパスを返却する
- `message` には内部例外メッセージではなく、外部公開可能なメッセージを返却する
- フロント側は `status`, `errorCode`, `message`, `details` を使って画面表示を制御する

<summary>4. HTTPステータス設計</summary>

| HTTPステータス | 用途 | 代表例 | 返却対象 | 備考 |
|---|---|---|---|---|
| 400 Bad Request | 入力値不正 | 必須不足、形式不正、状態不正 | 業務エラー | `details` を返却する場合あり |
| 401 Unauthorized | 未認証 / 認証失敗 | JWTなし、JWT不正、期限切れ、ログイン失敗 | 認証エラー | 保護APIでは再ログイン誘導対象 |
| 403 Forbidden | 権限不足 | 他人タスクの参照 / 更新 / 削除 | 認可エラー | 操作可否だけを通知する |
| 404 Not Found | 対象なし | タスク不存在、削除済みタスク、assignedUserId不存在 | 業務エラー | リソースが見つからない場合 |
| 409 Conflict | 状態競合・重複 | メールアドレス重複、更新競合 | 業務エラー | 登録済みメールアドレス、古い `version` での更新など |
| 500 Internal Server Error | システム障害 | DB障害、想定外例外 | システムエラー | 利用者向けメッセージは汎用化する |

<summary>8. バリデーションエラー設計</summary>

| 観点 | 方針 | レスポンス例 | フロントの扱い | ログレベル |
|---|---|---|---|---|
| 返却粒度 | field単位で details を返す | `[{"field":"title","message":"タイトルを入力してください"}]` | 各項目直下に表示 | WARN |
| message | messageは抽象、detailsで具体化 | `message: 入力内容に誤りがあります` | 全体表示 + 項目表示 | WARN |
| 複数件対応 | 複数項目エラーをまとめて返す | title / status / dueDate / priority / name / email / password | まとめて表示可能 | WARN |
| HTTP | 400 Bad Request | `status: 400` | フォーム再入力 | WARN |

## 8.1 バリデーションコード解決ルール

### Bean Validation

- TaskCreateRequest / TaskUpdateRequest の場合
  - `title` にエラーがある → `ERR-TASK-001`
  - `status` にエラーがある → `ERR-TASK-002`
  - `dueDate` にエラーがある → `ERR-TASK-003`
- 上記以外 → `ERR-INPUT-001`

### JSON変換エラー

- `status` の enum 不正 → `ERR-TASK-002`
- `dueDate` の形式不正 → `ERR-TASK-003`
- `priority` の enum 不正 → `ERR-INPUT-001`
- 上記以外 → `ERR-INPUT-001`

<summary>10. 画面側のエラー制御</summary>

## 10.1 認証画面

| 画面 | 主なエラー | 表示方法 |
|---|---|---|
| ログイン画面 | 入力不正、認証失敗 | 全体メッセージ + 項目エラー |
| 新規登録画面 | 入力不正、メール重複 | 全体メッセージ + 項目エラー |

## 10.2 タスク画面

| 画面 | 主なエラー | 表示方法 |
|---|---|---|
| タスク一覧画面 | 一覧取得失敗 | 画面上部エラーメッセージ |
| タスク詳細画面 | 詳細取得失敗、削除済みタスク取得、権限不足、削除失敗、更新競合 | 画面上部エラーメッセージ |
| タスク作成画面 | 入力不正、担当者未存在、認証エラー | 全体メッセージ + 項目エラー |
| タスク詳細画面内コメント編集 | コメント更新競合 | コメント欄エラーメッセージ + コメント一覧再取得案内 |
| 通知一覧画面 | 通知取得失敗、既読化失敗、関連タスク遷移失敗 | 画面上部エラーメッセージ |

## 10.3 401発生時の特別制御

- 保護APIで 401 を受けると、フロント側で認証情報をクリアする
- 保護パス上にいた場合は、再ログイン後の戻り先を保存する
- ログイン画面へ遷移し、再ログインを促すメッセージを表示する

<summary>14. 設計意図</summary>

| 観点 | 内容 |
|---|---|
| エラーコード | 利用するエラーコードを機能別に整理し、フロント・ログ・運用で同じ粒度で扱えるようにする |
| ユーザー体験 | `message` は抽象化し、`details` で具体的な入力エラーを補足することで、分かりやすさと安全性を両立する |
| 運用性 | `requestId` を返却し、問い合わせ時にログと突合できるようにする |
| セキュリティ | 認証失敗やシステム障害では内部情報を出しすぎず、安全なメッセージに統一する |
| 説明容易性 | ControllerAdvice、AuthenticationEntryPoint、AccessDeniedHandler、RequestIdFilter による一元化を説明しやすくする |

<summary>16. 備考</summary>

- 認証・認可エラーの一部は `GlobalExceptionHandler` ではなく Spring Security 側ハンドラで返却される
- `message` は外部公開可能な内容のみを返す
- フロントは主に `message` と `details` を用いて画面表示する
- 削除済みタスクは `ERR-TASK-004` に統一して未存在として扱う
- 将来エラーコードごとに画面文言や制御を分岐する場合は、本書を起点に拡張する
