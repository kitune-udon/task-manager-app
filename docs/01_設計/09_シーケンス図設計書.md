<summary>1. 文書概要</summary>

## 1.1 目的

本書は、task-manager-app における主要な画面操作およびAPI処理について、画面、フロントエンド状態管理、API呼び出し、バックエンド処理、DB / 外部ストレージ間の処理順序を Mermaid `sequenceDiagram` で定義する。

対象は以下とする。

- ログイン、新規登録、ログアウトなどの認証系操作
- ログイン後初期ロード、タスク一覧、タスク詳細、タスク作成、タスク更新、タスク削除
- チーム作成、所属チーム一覧、チーム詳細、チームメンバー一覧、チームメンバー追加、ロール変更、メンバー削除
- チーム文脈を持つタスク一覧、タスク詳細、タスク作成、タスク編集
- タスク詳細画面内のコメント、添付ファイル、アクティビティログ表示
- アプリ内通知一覧、未読通知件数取得、通知既読化、一括既読化
- 401発生時の再ログイン誘導、入力エラー応答などの共通例外系処理

<summary>2. 本書における設計前提</summary>


### 2.1 現フェーズで設計対象に含めるもの

本フェーズでは、**複数ユーザーで協業できるタスク管理アプリ** として、以下を現仕様に含める。

- チーム作成 / 所属チーム一覧 / チーム詳細 / チームメンバー管理
- チーム文脈を持つタスク一覧 / 詳細 / 作成 / 更新 / 削除
- タスク詳細画面内での編集
- タスク単位のコメント
- タスク単位の添付ファイル
- タスク単位のアクティビティログ
- アプリ内通知一覧と未読件数表示
- 通知既読化 / 一括既読化
- 添付ファイルの S3 保存
- タスク削除、コメント削除、添付削除の論理削除方針
- チーム内ロール `OWNER` / `ADMIN` / `MEMBER` に基づく操作制御

### 2.2 現フェーズで採用するUI/操作方針

以下を現仕様として固定する。

- コメント、添付、履歴は**タスク詳細画面に集約**する
- タスク編集は**詳細画面内の表示モード / 編集モード切替**で行う
- タスク削除は**物理削除ではなく論理削除**とする
- 通知は**アプリ内通知を中心**とし、メール通知やプッシュ通知は将来拡張とする
- 添付ファイルは**バックエンド経由でS3に保存 / 取得**する
- チーム一覧画面は `/teams`、チーム詳細画面は `/teams/{teamId}` とする
- `/tasks` は所属全チーム横断一覧、`/tasks?teamId={teamId}` はチーム文脈付き一覧として扱う
- タスク作成は `teamId` が確定したチーム文脈から行う
- タスク更新では所属チームを変更しない

### 2.3 将来構想として扱うもの

本書では、以下は将来拡張として扱い、現仕様のシーケンスには含めない。

- メール通知 / プッシュ通知 / リアルタイム通知
- コメントのリアルタイム更新
- 署名付きURLによる添付直接ダウンロード
- 添付ファイルの物理削除ジョブや高度なライフサイクル管理
- watcher / mention / チーム通知など通知対象の高度化
- チーム削除、招待リンク、メール招待、チーム単位の詳細権限設定

### 2.4 「チーム向け」の扱い

現フェーズでは、`teams`、`team_members`、`tasks.team_id` を前提に、**チーム境界を持つタスク管理アプリ** として整理する。  
タスク、コメント、添付、履歴は対象タスクの所属チームを起点に認可判定を行い、通知は通知受信者本人基準で参照可否を判定する。

  <details>
  <summary>5.1 SEQ-01 ログイン</summary>


  ### 概要

  利用者がログイン画面でメールアドレスとパスワードを入力し、認証に成功した場合、JWTと表示ユーザー名を保存してタスク一覧画面へ遷移する。

  ```mermaid
  sequenceDiagram
      autonumber
      actor User as 利用者
      participant Screen as ログイン画面
      participant AuthState as useAuthState
      participant AuthApi as authApi
      participant ApiClient as apiClient
      participant ReqFilter as RequestIdFilter
      participant AuthController as AuthController
      participant AuthService as AuthService
      participant UserRepo as UserRepository
      participant DB as users
      participant Storage as localStorage

      User->>Screen: メールアドレス・パスワード入力
      User->>Screen: ログイン押下
      Screen->>AuthState: handleLogin(event)
      AuthState->>AuthState: 入力チェック
      alt 入力エラーあり
          AuthState-->>Screen: エラーメッセージ表示
      else 入力エラーなし
          AuthState->>AuthApi: login(email, password)
          AuthApi->>ApiClient: POST /api/auth/login
          ApiClient->>ReqFilter: HTTPリクエスト
          ReqFilter->>AuthController: requestId設定後に処理継続
          AuthController->>AuthService: login(LoginRequest)
          AuthService->>UserRepo: findByEmail(email)
          UserRepo->>DB: ユーザー検索
          DB-->>UserRepo: User
          UserRepo-->>AuthService: User
          AuthService->>AuthService: パスワード照合
          AuthService->>AuthService: JWT生成
          AuthService-->>AuthController: LoginResponse
          AuthController-->>ReqFilter: 200 OK
          ReqFilter-->>ApiClient: LoginResponse
          ApiClient-->>AuthApi: response.data
          AuthApi-->>AuthState: token, user
          AuthState->>Storage: authToken保存
          AuthState->>Storage: userDisplayName保存
          AuthState->>Storage: postLoginRedirectPath取得・削除
          AuthState-->>Screen: /tasks または保存済み遷移先へ遷移
      end
  ```

  <summary>5.2 SEQ-02 新規登録</summary>


  ### 概要

  利用者が新規登録画面でユーザー情報を入力し、登録に成功した場合、ログイン画面へ戻す。

  ```mermaid
  sequenceDiagram
      autonumber
      actor User as 利用者
      participant Screen as 新規登録画面
      participant AuthState as useAuthState
      participant AuthApi as authApi
      participant ApiClient as apiClient
      participant ReqFilter as RequestIdFilter
      participant AuthController as AuthController
      participant AuthService as AuthService
      participant UserRepo as UserRepository
      participant DB as users

      User->>Screen: 名前・メールアドレス・パスワード入力
      User->>Screen: 登録する押下
      Screen->>AuthState: handleRegister(event)
      AuthState->>AuthState: 入力チェック
      alt 入力エラーあり
          AuthState-->>Screen: エラーメッセージ表示
      else 入力エラーなし
          AuthState->>AuthApi: register(name, email, password)
          AuthApi->>ApiClient: POST /api/auth/register
          ApiClient->>ReqFilter: HTTPリクエスト
          ReqFilter->>AuthController: requestId設定後に処理継続
          AuthController->>AuthService: register(RegisterRequest)
          AuthService->>UserRepo: existsByEmail(email)
          UserRepo->>DB: メールアドレス重複確認
          DB-->>UserRepo: exists結果
          UserRepo-->>AuthService: exists結果
          alt メールアドレス重複あり
              AuthService-->>AuthController: ConflictException
              AuthController-->>ApiClient: 409 ErrorResponse
              ApiClient-->>AuthApi: error
              AuthApi-->>AuthState: error
              AuthState-->>Screen: エラーメッセージ表示
          else メールアドレス重複なし
              AuthService->>AuthService: パスワードハッシュ化
              AuthService->>UserRepo: save(User)
              UserRepo->>DB: users追加
              DB-->>UserRepo: 保存結果
              UserRepo-->>AuthService: User
              AuthService-->>AuthController: RegisterResponse
              AuthController-->>ReqFilter: 201 Created
              ReqFilter-->>ApiClient: RegisterResponse
              ApiClient-->>AuthApi: response.data
              AuthApi-->>AuthState: 登録結果
              AuthState-->>Screen: 成功メッセージ設定
              AuthState-->>Screen: /login へ遷移
          end
      end
  ```

### 10.1 権限判定

| 操作 | 権限条件 |
|---|---|
| チーム作成 | 認証済みユーザー |
| 所属チーム一覧取得 | 認証済みユーザー。自分が所属するチームのみ |
| チーム詳細取得 | 対象チームに所属しているユーザー |
| チーム所属メンバー一覧取得 | 対象チームに所属しているユーザー |
| チーム追加候補ユーザー一覧取得 | 対象チームの `OWNER` / `ADMIN` |
| チームメンバー追加 | 対象チームの `OWNER` / `ADMIN` |
| チームメンバーのロール変更 | 対象チームの `OWNER` |
| チームメンバー削除 | 対象チームの `OWNER` / `ADMIN`。ただし `OWNER` 削除不可、担当中タスクありは削除不可 |
| タスク一覧取得 | ログインユーザーが所属するチームのタスクのみ取得。`teamId` 指定時は対象チーム所属が必要 |
| タスク詳細取得 | 対象タスクの所属チームにログインユーザーが所属していること |
| タスク作成 | 指定 `teamId` にログインユーザーが所属していること |
| タスク更新 | 対象タスクの所属チームに所属し、作成者 / 担当者 / `OWNER` / `ADMIN` のいずれか |
| タスク削除 | 対象タスクの所属チームに所属し、作成者 / `OWNER` / `ADMIN` のいずれか |
| コメント投稿 | 親タスクの所属チームに所属しているユーザー |
| コメント更新 / 削除 | コメント投稿者本人または親タスク所属チームの `OWNER` / `ADMIN` |
| 添付一覧 / ダウンロード | 親タスクの所属チームに所属しているユーザー |
| 添付アップロード | 親タスクを更新可能なユーザー |
| 添付削除 | 添付登録者本人または親タスク所属チームの `OWNER` / `ADMIN` |
| 通知一覧 / 既読化 | 自分宛通知のみ |
| 履歴一覧取得 | 親タスクの所属チームに所属しているユーザー |
