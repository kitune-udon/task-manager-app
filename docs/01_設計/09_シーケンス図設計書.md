## 3. 共通登場要素

| 要素 | 役割 |
|---|---|
| 利用者 | 画面を操作するユーザー |
| 画面 | React の各画面コンポーネント |
| useAuthState | 認証状態、ログイン、新規登録、ログアウトを管理する |
| useTaskState | タスク一覧、詳細、作成、編集、削除、担当者候補を管理する |
| authApi | 認証API呼び出しを行うフロント側APIモジュール |
| taskApi | タスクAPI呼び出しを行うフロント側APIモジュール |
| userApi | ユーザー候補取得API呼び出しを行うフロント側APIモジュール |
| apiClient | axiosクライアント。JWT付与と401検知を行う |
| localStorage | 認証トークン、表示ユーザー名、ログイン後遷移先を保持する |
| RequestIdFilter | requestIdを採番し、アクセスログを出力する |
| JwtAuthenticationFilter | JWTを検証し、SecurityContextへ認証情報を設定する |
| Controller | APIリクエストを受け付ける |
| Service | 業務処理、権限判定、DTO変換を行う |
| Repository | DBアクセスを行う |
| DB | PostgreSQL。users、tasksを保持する |

## 4. 認証系シーケンス

## 4.1 SEQ-01 ログイン

### 概要

利用者がログイン画面でメールアドレスとパスワードを入力し、認証に成功した場合、JWTと表示ユーザー名を保存してタスク一覧画面へ遷移する。

```mermaid
sequenceDiagram
    autonumber
    actor User as 利用者
    participant Screen as ログイン画面
    participant AuthState as useAuthState
    participant AuthApi as authApi
    participant ApiClient as apiClient
    participant ReqFilter as RequestIdFilter
    participant AuthController as AuthController
    participant AuthService as AuthService
    participant UserRepo as UserRepository
    participant DB as users
    participant Storage as localStorage

    User->>Screen: メールアドレス・パスワード入力
    User->>Screen: ログイン押下
    Screen->>AuthState: handleLogin(event)
    AuthState->>AuthState: 入力チェック
    alt 入力エラーあり
        AuthState-->>Screen: エラーメッセージ表示
    else 入力エラーなし
        AuthState->>AuthApi: login(email, password)
        AuthApi->>ApiClient: POST /api/auth/login
        ApiClient->>ReqFilter: HTTPリクエスト
        ReqFilter->>AuthController: requestId設定後に処理継続
        AuthController->>AuthService: login(LoginRequest)
        AuthService->>UserRepo: findByEmail(email)
        UserRepo->>DB: ユーザー検索
        DB-->>UserRepo: User
        UserRepo-->>AuthService: User
        AuthService->>AuthService: パスワード照合
        AuthService->>AuthService: JWT生成
        AuthService-->>AuthController: LoginResponse
        AuthController-->>ReqFilter: 200 OK
        ReqFilter-->>ApiClient: LoginResponse
        ApiClient-->>AuthApi: response.data
        AuthApi-->>AuthState: token, user
        AuthState->>Storage: authToken保存
        AuthState->>Storage: userDisplayName保存
        AuthState->>Storage: postLoginRedirectPath取得・削除
        AuthState-->>Screen: /tasks へ遷移
    end
```

### 主な正常終了条件

- JWTが返却されている
- `authToken` が保存される
- `userDisplayName` が保存される
- `/tasks` または保存済み遷移先へ遷移する

### 主な異常系

| 条件 | 挙動 |
|---|---|
| メールアドレス未入力 | 画面に項目エラーを表示 |
| メールアドレス形式不正 | 画面に項目エラーを表示 |
| パスワード未入力 | 画面に項目エラーを表示 |
| 認証失敗 | 401エラーを受け取り、画面にエラーメッセージを表示 |
| token未返却 | 画面に「トークンの取得に失敗しました。」を表示 |

## 4.2 SEQ-02 新規登録

### 概要

利用者が新規登録画面でユーザー情報を入力し、登録に成功した場合、ログイン画面へ戻す。

```mermaid
sequenceDiagram
    autonumber
    actor User as 利用者
    participant Screen as 新規登録画面
    participant AuthState as useAuthState
    participant AuthApi as authApi
    participant ApiClient as apiClient
    participant ReqFilter as RequestIdFilter
    participant AuthController as AuthController
    participant AuthService as AuthService
    participant UserRepo as UserRepository
    participant DB as users

    User->>Screen: 名前・メールアドレス・パスワード入力
    User->>Screen: 登録する押下
    Screen->>AuthState: handleRegister(event)
    AuthState->>AuthState: 入力チェック
    alt 入力エラーあり
        AuthState-->>Screen: エラーメッセージ表示
    else 入力エラーなし
        AuthState->>AuthApi: register(name, email, password)
        AuthApi->>ApiClient: POST /api/auth/register
        ApiClient->>ReqFilter: HTTPリクエスト
        ReqFilter->>AuthController: requestId設定後に処理継続
        AuthController->>AuthService: register(RegisterRequest)
        AuthService->>UserRepo: existsByEmail(email)
        UserRepo->>DB: メールアドレス重複確認
        DB-->>UserRepo: exists結果
        UserRepo-->>AuthService: exists結果
        alt メールアドレス重複あり
            AuthService-->>AuthController: ConflictException
            AuthController-->>ApiClient: 409 ErrorResponse
            ApiClient-->>AuthApi: error
            AuthApi-->>AuthState: error
            AuthState-->>Screen: エラーメッセージ表示
        else メールアドレス重複なし
            AuthService->>AuthService: パスワードハッシュ化
            AuthService->>UserRepo: save(User)
            UserRepo->>DB: users追加
            DB-->>UserRepo: 保存結果
            UserRepo-->>AuthService: User
            AuthService-->>AuthController: RegisterResponse
            AuthController-->>ReqFilter: 201 Created
            ReqFilter-->>ApiClient: RegisterResponse
            ApiClient-->>AuthApi: response.data
            AuthApi-->>AuthState: 登録結果
            AuthState-->>Screen: 成功メッセージ設定
            AuthState-->>Screen: /login へ遷移
        end
    end
```

### 主な正常終了条件

- users に新規ユーザーが登録される
- パスワードはハッシュ化される
- ログイン画面へ遷移する
- 登録済みメールアドレスがログインフォームへ反映される

## 5.2 SEQ-05 タスク一覧取得

### 概要

タスク一覧画面表示時または再読込時、ログインユーザーが参照可能なタスク一覧を取得する。

```mermaid
sequenceDiagram
    autonumber
    actor User as 利用者
    participant Screen as タスク一覧画面
    participant TaskState as useTaskState
    participant TaskApi as taskApi
    participant ApiClient as apiClient
    participant JwtFilter as JwtAuthenticationFilter
    participant TaskController as TaskController
    participant TaskService as TaskService
    participant CurrentUser as CurrentUserProvider
    participant TaskRepo as TaskRepository
    participant DB as tasks

    User->>Screen: 一覧表示または再読込
    Screen->>TaskState: loadTasks()
    TaskState->>TaskApi: fetchTasks()
    TaskApi->>ApiClient: GET /api/tasks
    ApiClient->>JwtFilter: Authorizationヘッダー付きリクエスト
    JwtFilter->>JwtFilter: JWT検証
    JwtFilter->>TaskController: 認証済みリクエスト
    TaskController->>TaskService: getTasks(status, priority, assignedUserId, keyword)
    TaskService->>CurrentUser: getCurrentUserId()
    CurrentUser-->>TaskService: currentUserId
    TaskService->>TaskRepo: searchAccessible(currentUserId, conditions)
    TaskRepo->>DB: 参照可能タスク検索

    TaskService->>TaskService: TaskSummaryResponseへ変換
    TaskService-->>TaskController: TaskSummaryResponse[]
    TaskController-->>ApiClient: 200 OK
    ApiClient-->>TaskApi: response.data
    TaskApi-->>TaskState: TaskItem[]
    TaskState->>TaskState: tasks更新
    TaskState-->>Screen: 一覧表示
```

### 補足

- 参照可能条件は「作成者本人」または「担当者本人」
- 画面上のステータス・優先度絞り込みはフロント側で実施する

## 5.4 SEQ-07 タスク作成

### 概要

タスク作成画面で入力内容を検証し、正常な場合はタスクを登録する。

```mermaid
sequenceDiagram
    autonumber
    actor User as 利用者
    participant Screen as タスク作成画面
    participant TaskState as useTaskState
    participant TaskApi as taskApi
    participant ApiClient as apiClient
    participant JwtFilter as JwtAuthenticationFilter
    participant TaskController as TaskController
    participant TaskService as TaskService
    participant CurrentUser as CurrentUserProvider
    participant UserRepo as UserRepository
    participant TaskRepo as TaskRepository
    participant UsersDB as users
    participant TasksDB as tasks

    User->>Screen: タスク情報入力
    User->>Screen: タスクを作成押下
    Screen->>TaskState: handleCreateTask(event)
    TaskState->>TaskState: 入力チェック
    alt 入力エラーあり
        TaskState-->>Screen: エラーメッセージ表示
    else 入力エラーなし
        TaskState->>TaskApi: createTask(CreateTaskRequest)
        TaskApi->>ApiClient: POST /api/tasks
        ApiClient->>JwtFilter: Authorizationヘッダー付きリクエスト
        JwtFilter->>JwtFilter: JWT検証
        JwtFilter->>TaskController: 認証済みリクエスト
        TaskController->>TaskService: createTask(TaskCreateRequest)
        TaskService->>UserRepo: findById(assignedUserId)
        UserRepo->>UsersDB: 担当者検索
        UsersDB-->>UserRepo: Userまたはnull
        UserRepo-->>TaskService: assignedUser
        TaskService->>CurrentUser: getCurrentUserId()
        CurrentUser-->>TaskService: currentUserId
        TaskService->>UserRepo: findById(currentUserId)
        UserRepo->>UsersDB: 作成者検索
        UsersDB-->>UserRepo: User
        UserRepo-->>TaskService: createdBy
        TaskService->>TaskRepo: save(Task)
        TaskRepo->>TasksDB: tasks追加
        TasksDB-->>TaskRepo: 保存結果
        TaskRepo-->>TaskService: Task
        TaskService->>TaskService: TaskResponseへ変換
        TaskService-->>TaskController: TaskResponse
        TaskController-->>ApiClient: 201 Created
        ApiClient-->>TaskApi: response.data
        TaskApi-->>TaskState: TaskItem
        TaskState->>TaskState: loadTasks()
        TaskState-->>Screen: 一覧画面へ遷移
    end
```

### 補足

- `createdBy` はログインユーザーから設定される
- `assignedUserId` が未指定の場合、担当者なしで登録される
- 成功後は一覧を再取得し、成功メッセージを表示する

## 5.5 SEQ-08 タスク編集

### 概要

タスク編集画面で既存値をフォームに反映し、更新押下時にタスクを更新する。

```mermaid
sequenceDiagram
    autonumber
    actor User as 利用者
    participant DetailScreen as タスク詳細画面
    participant EditScreen as タスク編集画面
    participant TaskState as useTaskState
    participant TaskApi as taskApi
    participant ApiClient as apiClient
    participant JwtFilter as JwtAuthenticationFilter
    participant TaskController as TaskController
    participant TaskService as TaskService
    participant CurrentUser as CurrentUserProvider
    participant UserRepo as UserRepository
    participant TaskRepo as TaskRepository
    participant UsersDB as users
    participant TasksDB as tasks

    User->>DetailScreen: 編集押下
    DetailScreen->>TaskState: handleShowEdit()
    TaskState-->>EditScreen: 編集画面へ遷移
    TaskState->>TaskState: selectedTaskをeditFormへ反映

    User->>EditScreen: タスク情報変更
    User->>EditScreen: 更新する押下
    EditScreen->>TaskState: handleEditTask(event)
    TaskState->>TaskState: 入力チェック
    alt 入力エラーあり
        TaskState-->>EditScreen: エラーメッセージ表示
    else 入力エラーなし
        TaskState->>TaskApi: updateTask(taskId, UpdateTaskRequest)
        TaskApi->>ApiClient: PUT /api/tasks/:id
        ApiClient->>JwtFilter: Authorizationヘッダー付きリクエスト
        JwtFilter->>JwtFilter: JWT検証
        JwtFilter->>TaskController: 認証済みリクエスト
        TaskController->>TaskService: updateTask(taskId, TaskUpdateRequest)
        TaskService->>CurrentUser: getCurrentUserId()

        TaskService->>TaskRepo: findWithAssignedUserById(taskId)
        TaskRepo->>TasksDB: 既存タスク取得
        TasksDB-->>TaskRepo: Task
        TaskRepo-->>TaskService: Task
        TaskService->>TaskService: 更新権限判定
        alt 更新権限なし
            TaskService-->>TaskController: BusinessException
            TaskController-->>ApiClient: 403 ErrorResponse
            ApiClient-->>TaskState: error
            TaskState-->>EditScreen: エラーメッセージ表示
        else 更新権限あり
            TaskService->>UserRepo: findById(assignedUserId)
            UserRepo->>UsersDB: 担当者検索
            UsersDB-->>UserRepo: Userまたはnull
            UserRepo-->>TaskService: assignedUser
            TaskService->>TaskService: 変更項目判定
            TaskService->>TaskRepo: save(Task)
            TaskRepo->>TasksDB: tasks更新
            TasksDB-->>TaskRepo: 更新結果
            TaskRepo-->>TaskService: Task
            TaskService-->>TaskController: TaskResponse
            TaskController-->>ApiClient: 200 OK
            ApiClient-->>TaskApi: response.data
            TaskApi-->>TaskState: TaskItem
            TaskState->>TaskState: loadTasks()
            TaskState-->>DetailScreen: 詳細画面へ遷移
        end
    end
```

### 補足

- 更新可能条件は「作成者本人」または「担当者本人」
- 成功後は詳細画面へ戻る
- 更新後に一覧も再取得する

## 6. ユーザー候補取得シーケンス

## 6.1 SEQ-10 担当者候補取得

### 概要

タスク作成・編集画面の担当者プルダウンで使用するユーザー一覧を取得する。

```mermaid
sequenceDiagram
    autonumber
    participant TaskState as useTaskState
    participant UserApi as userApi
    participant ApiClient as apiClient
    participant JwtFilter as JwtAuthenticationFilter
    participant UserController as UserController
    participant UserService as UserService
    participant UserRepo as UserRepository
    participant DB as users

    TaskState->>UserApi: fetchAssignableUsers()
    UserApi->>ApiClient: GET /api/users
    ApiClient->>JwtFilter: Authorizationヘッダー付きリクエスト
    JwtFilter->>JwtFilter: JWT検証
    JwtFilter->>UserController: 認証済みリクエスト
    UserController->>UserService: getUsers()
    UserService->>UserRepo: findAll(name ASC, id ASC)
    UserRepo->>DB: ユーザー一覧検索
    DB-->>UserRepo: User[]
    UserRepo-->>UserService: User[]
    UserService->>UserService: UserResponseへ変換
    UserService-->>UserController: UserResponse[]
    UserController-->>ApiClient: 200 OK
    ApiClient-->>UserApi: response.data
    UserApi-->>TaskState: AssignableUser[]
    TaskState->>TaskState: assigneeOptions生成
```

### 補足

- 先頭に `未選択` が追加される
- 編集対象の担当者が候補に含まれない場合は、選択肢に補完される

## 7. 認証エラー・例外系シーケンス

## 7.1 SEQ-11 401発生時の再ログイン誘導

### 概要

保護APIで 401 が返却された場合、フロント側で認証情報をクリアし、ログイン画面へ戻す。

```mermaid
sequenceDiagram
    autonumber
    participant Screen as 保護画面
    participant TaskState as useTaskState
    participant TaskApi as taskApi
    participant ApiClient as apiClient
    participant Storage as localStorage
    participant Window as window
    participant AuthState as useAuthState

    Screen->>TaskState: API実行
    TaskState->>TaskApi: fetch / create / update / delete
    TaskApi->>ApiClient: 保護API呼び出し
    ApiClient-->>ApiClient: 401レスポンス受信
    alt 認証API以外
        ApiClient->>Storage: authToken削除
        ApiClient->>Window: app:unauthorizedイベント発火
        Window->>AuthState: handleUnauthorized()
        AuthState->>Storage: 必要に応じてpostLoginRedirectPath保存
        AuthState->>Storage: userDisplayName削除
        AuthState->>AuthState: token / currentUserLabelクリア
        AuthState-->>Screen: /login へ遷移
    else 認証API
        ApiClient-->>TaskApi: 通常のエラーとして返却
    end
```

### 補足

- `/api/auth/` 配下の 401 では `app:unauthorized` を発火しない
- 保護パスにいた場合は、再ログイン後の戻り先としてパスを保存する

## 7.2 SEQ-12 入力エラー応答

### 概要

画面側の入力チェックまたはAPI側のバリデーションにより、入力エラーを画面へ表示する。

```mermaid
sequenceDiagram
    autonumber
    actor User as 利用者
    participant Screen as 入力画面
    participant State as useAuthState / useTaskState
    participant Api as authApi / taskApi
    participant ApiClient as apiClient
    participant Controller as Controller
    participant ExceptionHandler as GlobalExceptionHandler

    User->>Screen: 入力して送信
    Screen->>State: submit handler
    State->>State: ローカル入力チェック
    alt ローカル入力エラーあり
        State->>State: fieldErrors設定
        State-->>Screen: 項目エラー表示
    else ローカル入力エラーなし
        State->>Api: API呼び出し
        Api->>ApiClient: HTTPリクエスト
        ApiClient->>Controller: request
        Controller->>Controller: Bean Validation
        alt API入力エラーあり
            Controller-->>ExceptionHandler: MethodArgumentNotValidException
            ExceptionHandler-->>ApiClient: 400 ErrorResponse
            ApiClient-->>Api: error
            Api-->>State: error
            State->>State: fieldErrors設定
            State-->>Screen: 項目エラー表示
        else API入力エラーなし
            Controller-->>ApiClient: 正常レスポンス
            ApiClient-->>Api: response
            Api-->>State: result
            State-->>Screen: 成功時の画面更新
        end
    end
```

## 8. 補足事項

### 8.1 権限判定

| 操作 | 権限条件 |
|---|---|
| タスク一覧取得 | 作成者または担当者として参照可能なタスクのみ取得 |
| タスク詳細取得 | 作成者または担当者のみ参照可能 |
| タスク作成 | 認証済みユーザーであれば作成可能 |
| タスク更新 | 作成者または担当者のみ更新可能 |
| タスク削除 | 作成者のみ削除可能 |

### 8.2 フロント側の主な状態更新

| 処理 | 更新される状態 |
|---|---|
| ログイン成功 | token, currentUserLabel |
| ログアウト | token, currentUserLabel, tasks, selectedTask, form, message |
| タスク一覧取得 | tasks |
| タスク詳細取得 | selectedTask |
| タスク作成 | createForm, tasks, successMessage |
| タスク更新 | selectedTask, tasks, successMessage |
| タスク削除 | selectedTask, tasks, successMessage |
| 担当者候補取得 | assignableUsers, assigneeOptions |

### 8.3 localStorage 利用項目

| キー | 用途 |
|---|---|
| authToken | JWT |
| userDisplayName | ヘッダー表示名 |
| postLoginRedirectPath | 再ログイン後の戻り先 |
